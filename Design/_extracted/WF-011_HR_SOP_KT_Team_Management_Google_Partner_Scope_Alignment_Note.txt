1. Purpose of This Note
This note is meant to help the Google Partner / implementation partner align their technical scope with the intended HKHT AI for Seva architecture for WF-011 HR / SOP / KT / Team Management.
This is not a rejection of the current partner work.
This is not a request to add random new scope.
This is a clarification of the intended architecture so the partner can classify, phase and strengthen its plan correctly.
The current workflow is only evidence of today’s reality. The target is an AI-first, agentic, CRM-aligned, ERP-ready, dashboard-visible and human-governed platform module.
Evidence basis: Based on Latest Document B (S1), Transformation Analysis AI-first redesign principle (S8), GE Gap Analysis (S5).
2. Core Understanding Expected from the Partner
WF-011 should become the HR capacity, SOP, KT, role ownership and institutional-memory module of the Hyderabad AI Revenue & Seva Growth Platform.
It should not be reduced to a small isolated demo.
It should not become a disconnected automation.
It should not be treated only as a knowledge assistant over SOPs and HR policies.
It should be a production-aligned module that manages roles, owners, backups, onboarding, attendance exceptions, KT completion, performance pulse and leadership visibility.
It must preserve common IDs, source tracking, approvals, dashboards, CRM/ERP readiness and future multi-center scale.
It must keep private HR operational records within HR/ERP/HRMS boundaries while exposing only necessary capacity/assignment signals to revenue workflows.
Evidence basis: Based on Latest Document B platform role (S1), GE Technical Scope WF-011 classification (S4), GE Gap Analysis WF-011 concerns (S5).

7. What the Partner Should Not Miss
Common IDs: Employee ID, Role ID, Owner ID, Task ID, Approval ID, SOP ID, Training ID and, where capacity touches relationship work, Contact/Donor/Yatri/Campaign/Event IDs.
CRM is relationship source of truth; HR private records must remain within HR/ERP/HRMS boundaries.
ERP-ready fields must be captured from Day 1 even if ERP build happens later.
Approval gates are mandatory for hiring, salary, payroll, incentives, deductions, discipline, termination, final reviews and sensitive communication.
Audit logs must show who approved, when and what was approved.
Role ownership validation is required before routing is finalized.
Fallback process is mandatory for missing API or unreliable data source.
API registry should list HRMS, biometric, ClickUp/Wrike, ATS, payroll, CRM and ERP dependencies.
Golden journey testing must prove vacancy → JD → onboarding → SOP/KT → attendance → backup → dashboard.
Leadership command center contribution must be explicit.
KCKE and Media AI boundaries must be respected.
WF-013 future packaging should be considered in templates and metadata.
