Production Readiness Definition
For WF-003, production readiness does not mean fully autonomous ad optimization. It means the workflow can be used in live operations by selected HKHT users with validated data flow, visible dashboards, human approvals, fallback path, and measurable productivity improvement.
Minimum production-readiness criteria for this workflow:
Live operational usage for at least one active or recent fundraising/Yatra/donation campaign.
Validated campaign data path: campaign intake -> Campaign ID/source tag -> landing page/form/payment path -> CRM/HelloLeads record -> follow-up owner -> dashboard output.
Dashboard visibility for spend, leads, CPL, donations, Yatra registrations, revenue, ROI, pending approvals, failed handoffs and exceptions.
Human approval routing for budget changes, public copy, donor-sensitive communication, devotional/philosophical content, media/deity assets and high-risk exceptions.
Role ownership validated: owner, performer, approver, backup, escalation person, dashboard reviewer and data custodian.
Data quality checks for mandatory IDs, source tags, duplicate leads, suppression/DND flags, missing payment attribution, stale follow-ups and failed lead sync.
AI-assisted execution for campaign brief, content variants, audience suggestions, dashboard interpretation and campaign learning summaries.
Cross-workflow compatibility with WF-001 Yatra Lifecycle, WF-002 Voice Agent + CRM Follow-up, WF-006 CRM/DBMS, WF-007 Donor Cultivation, WF-009 Finance/DCC/P&L, WF-012 Landing Pages and WF-014 Community Outreach.
Fallback process if API, WhatsApp/RCS/SMS, CRM, payment, dashboard or agentic automation fails during a live campaign.
Golden journey test: a sample contact enters through a WF-003 campaign and can be traced through CRM follow-up, donation or Yatra registration, payment/receipt status and next action.

API dependency tracking: avoid promising live integrations until access, provider, authentication, limits, costs and refresh cadence are confirmed in the API Registry.
Fallback path: if automation fails, the workflow must continue through controlled Sheets/export/import/manual approval with clear owner and escalation logic.
